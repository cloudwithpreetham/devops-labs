# Day 05 – Object-Oriented Python for DevOps

## Objective

The goal of Day 05 was to refactor the Day 04 log analyzer script using a simple Object-Oriented Programming approach.

Instead of keeping all logic in a single procedural script, the log analysis functionality was organized inside a Python class. This makes the script cleaner, easier to maintain, and closer to how real-world DevOps automation tools are structured.

---

## Task Summary

For this task, I created an OOP-based log analyzer that:

- Reads an application log file
- Counts different log levels:
  - `INFO`
  - `WARNING`
  - `ERROR`

- Prints the summary in the terminal
- Writes the summary to an output file
- Uses a class-based structure with methods

The script contains a `LogAnalyzer` class that handles:

- Running the complete workflow

## Why OOP Was Used

In DevOps, automation scripts often grow over time. A small script may later include more features such as:

- Multiple log files
- JSON reports
- Email alerts
- Monitoring integration
- Error filtering
- Log severity dashboards

Using a class helps keep related data and behavior together. This makes the script easier to read, debug, and extend.

---

## What I Learned

Today I learned how to refactor a simple Python script using Object-Oriented Programming.

Key takeaways:

- A class helps group related logic together
- `__init__()` is used to initialize values when an object is created
- Methods make the script more organized and reusable
- OOP improves readability when automation scripts become larger
- Simple OOP is enough for many real-world DevOps tools

Log analysis is a common DevOps activity. Engineers often need to check logs to understand application health, troubleshoot failures, and monitor system behavior.

This project demonstrates a basic version of how internal DevOps tools can be structured using Python classes.
